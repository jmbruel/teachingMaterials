    if(a > b) {

        return false;
    }
}

== Why ?

[%step]
- Improve *understanding* of code
- Help find and fix *bugs*
- Accelerate the *speed* of development
- Improve *design*

== Code smells (basic)

[%step]
- Large class
- Long methods
- Too many parameters
- Using a lot of primitive data types (`String`, `float`, ...)

== Code smells (OO)

[%step]
- Switch case
- Temporary field (`null`)
- Parallel inheritance hierarchies
- Wrong use of inheritance

== Garbage Smells

[%step]
- Too many comments
- Duplicated code
- Lazy class
- Unused code

== Excessive connectivity

[%step]
- External methods
- Dependency with another class' implementation details
- Long class call

== Extract a method

[source,java]

public void calcQuadraticEq(double a, double b, double c) {
    double D = b * b - 4 * a * c;
    if (D > 0) {
        double x1, x2;
        x1 = (-b - Math.sqrt(D)) / (2 * a);
        x2 = (-b + Math.sqrt(D)) / (2 * a);
        System.out.println("x1 = " + x1 + ", x2 = " + x2);
    }
    else if (D == 0) {
        double x;
        x = -b / (2 * a);
        System.out.println("x = " + x);
    }
    else {
        System.out.println("Equation has no roots");
    }
}
